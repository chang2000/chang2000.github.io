# 放弃使用Emacs ox-hugo管理博客文件



从接触GNU Emacs以及自由软件文化以来，自己就一直致力于把自己的工作流整合到Emacs这一个单纯的软件里。可当我真正整合之后，事情好像也没有我想的那么简单。

有一句话叫没那个金刚钻就别揽那个瓷器活儿。很多人，当然包括我，在探寻技术的道路上兜兜转转，把过多的精力放到了我应该使用什么工具上，我应该把我的工具打造成什么样子，可还是那句老话，一个好的铁匠并不一定是一个好的剑客，要时刻提防自己在岔路口越走越远。

好像有一点儿跑题，上面的陈述只是表达了我对工具和技术之间关系的看法，不一定对每个人都适用。那下面就来点儿具体的，为什么要放弃在Emacs里书写博客之类的呢，是因为Emacs不够好吗？

我在作出这个决定前也是有所考虑的，毕竟改变自己工作流是一件很繁琐的事情，关于有什么东西会把我一直留在Emacs里写博客呢：

- Org Mode本身的优越性
  我个人本身是org mode的超级拥趸，关于org mode的好我不在这里多言。org mode可以为写作提供各种便利，例如表格代码和各种大纲排版，但是自从我把博客的工作流尝试转向org导出成md后，事情并没有我想的那么美好。
- 工作流的一体化
  这里要提到一个具有创举性的项目：Emacs Application Framework. 这是由中国的Emacs Hacker Manateelazycat 开发的让你在Emacs里运行所有应用的框架。这是很具有诱惑性的。从此鼠标是路人的那种。

那为什么会考虑丢弃掉ox-hugo呢：

我承认自己并不是那种天生的hacker，技术领悟程度其实是不高的。为Emacs配置中文写作环境是一件非常麻烦的事情，甚至直接涉及到输入法的框架。再加上自己很少在书写博客的过程中现写代码，基本都是从自己的gist上粘贴下来，所以Emacs编辑代码的便捷也开始没有那么重要。再加上自己想专心对着一个像纸一样的界面进行思维输出，而不是很geek的桌面，他们影响了我的思维。优雅的Ulysses和Typora才是真正的distraction free。我是Mac使用者，全局的Emacs keybinding也让我在移动光标时可以不使用鼠标。

也许几年后我又会觉得现在的自己就是个傻波一，但是现在更重要的是get your work done.

我依然拥护Emacs，但我开始倾向于把它作为开发的集成平台，而不是我的全部生活，尽管Emacs永远不会死。我现在正在Ulysses里编辑这篇文章，我的心很沉浸，注意力全在我所输出的内容上，这种感觉很棒。
